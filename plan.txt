1. Зависимости
    Базовые
    node — runtime.

    yarn@1.22.22 — менеджер пакетов.

    typescript — типизация.

    vite — сборка и dev-сервер, актуальная стабильная ветка 8.x.

    xlsx — SheetJS для чтения/записи .xls, .xlsx, .csv; для публичного npm-источника ориентируйся минимум на 0.20.2.

    Для разработки
    @types/node — типы Node.js.

    vite-plugin-checker — если хочешь быстрые проверки в dev.

    eslint и prettier — по желанию, для чистоты кода.

    web-worker отдельно ставить не нужно, это встроенный API браузера.

    Команды установки
    bash
    yarn add xlsx
    yarn add -D typescript vite @types/node eslint prettier

4. Поток работы
    Пользователь выбирает файл A и файл B.

    main.ts проверяет расширения и отправляет файлы в worker.ts.

    worker.ts читает оба файла через xlsx.

    worker.ts преобразует таблицы в единый формат.

    worker.ts выполняет сведение и собирает итоговый массив.

    worker.ts генерирует итоговый файл.

    main.ts получает результат и отдает пользователю скачивание.

6. Производительность
    Все тяжёлые операции выполнять в Web Worker.

    Избегать лишнего состояния в UI.

    Не использовать тяжелую таблицу-обертку, если цель — только импорт, сведение и экспорт.

    Для очень больших файлов держать логику потоковой и минимизировать промежуточные копии данных.

9. Web Worker: чтение и общение
    Worker лучше строить так: UI отправляет в него File/ArrayBuffer, worker читает через xlsx, обрабатывает данные и отвечает сообщением с результатом. Для обмена делай postMessage с типизированными payload'ами: например { type: 'parse', fileA, fileB }, а в ответ { type: 'done', rows, blob } или { type: 'error', message }.
    Если таблицы большие, передавай в worker именно ArrayBuffer и используй transferables, чтобы не копировать память лишний раз; UI тогда только показывает прогресс и инициирует скачивание итогового Blob.

    Могу следующим сообщением сразу дать готовый шаблон package.json + vite.config.ts
